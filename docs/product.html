How It Works

Drop files in. Ask questions.
Get cited answers.

No training. No fine-tuning. No IT project. Your documents are chunked, embedded, and stored in a private vector index on your sole-tenant GCP node. Queries run on your private NVIDIA GPU node against a 123B parameter language model — nothing touches shared infrastructure.

Step 01

Drop Documents

PDF, DOC, DOCX, TXT, MD, RST, CSV, HTML, and images (PNG, JPG, JPEG, TIFF, BMP, WEBP). Drop into a watched folder or GCS bucket. Three-tier PDF extraction: native text → poppler → OCR. Images and scanned documents go directly to tesseract OCR.

Step 02

Automatic Ingest

The ra-watch daemon detects new files within seconds. Legal-aware chunker splits at section boundaries — not arbitrary character counts.

Step 03

Private Embedding

1024-dimension embeddings computed on-device by gte-large-en-v1.5 via ONNX Runtime with CUDA acceleration. Zero API calls.

Step 04

Post-Quantum Vault Encryption

Every document shard encrypted with AES-256-GCM. Keys wrapped with ML-KEM Kyber-768 (NIST FIPS 203). Data is decrypted only for the duration of a query — never written to disk in plaintext. No competitor does this.

Step 05

HNSW Vector Index

Vectors stored in a bare-metal HNSW index (usearch). Sub-millisecond approximate nearest-neighbor retrieval at any scale.

Step 06

Ask Your Vault

Natural language query → top-K retrieved chunks → Mistral Large 2 (123B) generates a cited answer. Every response shows the source document and paragraph.

Step 07

ML-DSA-65 Signed Output

Every answer and audit log entry is signed with ML-DSA-65 (NIST FIPS 204) — a quantum-resistant lattice signature. Tamper-evident chain of custody. Verifiable by anyone with our public key.
